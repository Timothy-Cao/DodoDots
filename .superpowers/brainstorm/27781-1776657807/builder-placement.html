Level Builder — how should users place nodes and edges?

Two interaction models. Pick the one that feels right for a Tron-style graph editor.

A

Free-placement mode switcher

Node mode: click empty canvas to drop a node. Click a node to select it (for count, start-eligible toggle).

Edge mode: click one node then another to create an edge. Click existing edge to select (toggle direction, set count).

Properties panel on the right shows details for the selected element with count (1–9) slider, direction toggle, start-eligible toggle.

Pros: Clean, few clicks. Cons: Must remember which mode you're in.

B

Modeless drag-to-connect

Double-click empty canvas to create a node. Drag from one node to another to create an edge. Click anything to select it.

All properties edited via the right-side inspector panel. Keyboard shortcuts: 1–9 sets count of selected element, D cycles edge direction, S toggles start-eligible, Delete removes.

Pros: No mode switching, fluid. Cons: Drag-to-connect can feel finicky with small nodes; double-click vs single-click needs care.

C

Hybrid: toolbar + drag

Top toolbar with three tools: Select, Node, Edge (keyboard: V / N / E — like Figma). In Node tool, click to drop. In Edge tool, drag between nodes. Select tool is default for editing properties.

Inspector panel on the right for selected item. Same keyboard shortcuts as B for properties.

Pros: Familiar to anyone who's used a design tool; predictable. Cons: One more click than B for quick additions.

In all three: global level settings (max moves, level name) live in a small panel. "Playtest" button transitions to the gameplay view running on the current graph. "Publish" is disabled until you've playtested successfully.
